Project Description

Chat Application

Based on Spring Monolithic Architecture


Introduction

The Chatting Application is a communication platform developed using the Java Spring Boot framework, employing the Spring Boot Monolithic architecture. This application aims to provide users with a seamless and comprehensive solution for real-time messaging, user profiles, and related functionalities. The project encompasses modules such as Chat Management, User Profiles, Message Sending, Notification Handling, and User Feedback.



Technology Stack

Programming Language : Java

Framework : Spring Boot

Architecture : Monolithic

Database : MySQL & MongoDB

Other Tools : Lombok,Swagger,Actuator,Kafka, WebSocket



Modules

Chat Management:Manages the list of chat conversations, including participants, messages, and timestamps.Handles group chats, one-on-one conversations, and any associated rules or restrictions.


User Profiles: Manages user information, including profiles, preferences, and historical chat data.Provides functionalities for user authentication and authorization.


Message Sending: Allows users to send text and multimedia messages in real-time.Facilitates the delivery and reception of messages with support for attachments and emojis.


User Feedback: Enables users to submit feedback on the application's usability and performance.Manages a repository of user feedback, contributing to continuous improvement.



Application Health Check Mechanism

The application has a health check mechanism implemented using Spring Boot Actuator. The Actuator module provides useful insights into the Spring environment for a running application with functions for health checking and metrics gathering. The health check endpoint is exposed over HTTP and JMX.



Monitoring and Observability

The monitoring and observability of the application are achieved using health check functions in Spring Boot applications and making them observable by capturing useful health metrics. The application integrates with popular monitoring tools for predicting system health in the future by observing any anomalies in a series of metrics like memory utilization, errors, and disk space.



Future Modules

Analytics Dashboard: Develops an analytics dashboard for monitoring the application's performance, user engagement, and messaging trends.


chat Group: Develops an chat Group for multiple users using websockets.



Conclusion

This Chatting Application is a robust and user-friendly solution, providing modules covering Chat Management, User Profiles, Message Sending, Notification Handling, and User Feedback. The incorporation of Spring Boot Actuator ensures health check mechanisms, making the application observable and monitorable. The envisioned future modules aim to further enrich the application's functionality and enhance the overall user experience in the realm of real-time communication.
