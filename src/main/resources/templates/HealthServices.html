Project Description

Healthcare Services Application

Based on Restful Microservices Architecture


Introduction

This project is a health services application built using Restful Microservices Architecture with Spring Boot and Java. The application has modules like Doctor, User, Insurance, Appointment Booking, Health Blog, Review Doctor, and Rate Doctor. The application provides functionalities for managing doctors, users, insurance, appointment booking, health blogs, doctor reviews, and doctor rating.



Technology Stack

Programming Language : Java

Framework : Spring Boot

Architecture : Microservices

Database : MySQL & MongoDB

Other Tools : Lombok, Eureka Server, Hystrix Circuit Breaker, Swagger, Webflux, Actuator, RestApi



Modules

Doctor: This module manages the list of doctors, their availability, and specializations. It allows users to search for doctors based on their specialization and availability.


User: This module manages the list of users and their details. It allows users to register, login, and update their personal information.


Insurance: This module manages the list of insurances and their details. It allows users to search for insurance providers based on their location and services.


Appointment Booking: This module allows users to book appointments with doctors. It provides a user-friendly interface for users to select a doctor, date, and time for their appointment.


Health Blog: This module contains health-related articles and blogs. It allows users to read articles and blogs related to health and well-being.


Review Doctor: This module allows users to review doctors based on their experience. It provides a platform for users to share their reviews and feedback about doctors.


Rate Doctor: This module allows users to rate doctors based on their experience. It provides a user-friendly interface for users to rate doctors on various factors such as professionalism, communication, and punctuality.



Health Check Mechanism

The application has a health check mechanism implemented using Spring Boot Actuator. The Actuator module provides useful insights into the Spring environment for a running application with functions for health checking and metrics gathering. The health check endpoint is exposed over HTTP and JMX.



Monitoring and Observability

The monitoring and observability of the application are achieved using health check functions in Spring Boot applications and making them observable by capturing useful health metrics. The application integrates with popular monitoring tools for predicting system health in the future by observing any anomalies in a series of metrics like memory utilization, errors, and disk space.



API Gateway

is a server that acts as an entry point into a system, handling all external API requests and routing them to the appropriate microservices. It can also perform tasks such as authentication, rate limiting, and caching.



Eureka Server

is a service registry that enables microservices to discover and communicate with each other. It provides a centralized location for microservices to register themselves, allowing them to be discovered by other microservices and the API Gateway.



Payment Processing

Integrates a payment gateway for secure and seamless transaction processing. Supports various payment methods and ensures the confidentiality of user financial information.



Future Modules

Prescription Management: This module can manage the prescriptions given by doctors to patients.


Chatbot Integration: This module can manage the chatbot integration for answering health-related queries.


Analytics Dashboard: This module can provide an analytics dashboard for monitoring the application's performance and usage.




Conclusion

This project is a comprehensive health services application with modules like Doctor, User, Insurance, Appointment Booking, Health Blog, Review Doctor, and Rate Doctor. The application has a health check mechanism implemented using Spring Boot Actuator, making it observable and monitorable. The future modules can enhance the application's functionality and user experience.
